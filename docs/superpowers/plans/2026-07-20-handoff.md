## 3. 环境备忘

### 3.1 go.work 陷阱

`legion/go.work` 的 use 列表**不含** `.claude/worktrees/*`。在 worktree 里所有 go 命令必须加 `GOWORK=off`，主工作区不用。

### 3.2 race detector

两套都可用，**不是二选一**：

- **Windows**：LLVM-MinGW 已在系统 PATH，新终端直接 `go test -race ./...`
- **WSL**：`export PATH=/opt/go/bin:$PATH GOWORK=off CGO_ENABLED=1 GOOS=linux GOARCH=amd64`

**`GOOS=linux` 不能省** —— WSL 的 go env 里 `GOOS='windows'`，不覆盖会报 `-mthreads` 错误（MinGW 选项，Linux gcc 不认）。

**涉及文件权限或符号链接的改动应以 WSL 结果为准**：`TestReadEventsFailsLoudOnUnreadableDirectory` 在 Windows 上是 `t.Skip`，symlink 测试在 Windows 需开发者模式。

### 3.3 变异测试是本仓库的惯例

新增测试后，把生产代码改回旧写法确认测试**确实失败**，再改回来。

本轮它抓到过两个假通过的测试（详见 `testing.md`）。这一步不能省。

### 两个曾经的误判（避免重蹈）

1. **日志里的 `agent_id` 字段**：成功事件里是实际执行的 agent（可信），**失败事件里是 coordinator 自身的 ID**（恒为 `default-agent`）。我曾据此误判「sub-agent 没生效」，实际生效了。已写进 `troubleshooting.md`。
